## 3. Decisions Made

| Decision | Rationale |
|----------|-----------|
| Treat empty-invocation help as a useful fallback, not as a primary agent discovery contract | The spec prioritizes machine-readable discovery such as `--schema` and manifest output over human-formatted help |
| Choose the lightweight documentation path instead of adding a new challenge or requirement | The behavior fit existing requirements around help purity, non-TTY safety, and schema discoverability well enough without expanding the top-level model |
| Keep root help lightweight and point agents to `--schema` | Automatic full schema output on bare invocation would be noisy and blur the distinction between fallback help and the explicit discovery interface |
| Treat broken-link findings under `docs/*` as OK | `docs/` is a symlinked publishing mirror, and the published site did not exhibit user-visible broken links |
| Fix only the actionable non-`docs/*` broken links | This addressed real stale references without churning copied or mirror-specific content |

## 4. Decisions Refused / Alternatives Rejected

| Rejected Approach | Reason |
|-------------------|--------|
| Add a standalone challenge for no-argument invocation behavior | The behavior was adjacent to existing requirements and did not yet justify a separate failure mode |
| Make no-argument invocation automatically return the full `--schema` payload in agent JSON mode | That would be heavier than necessary and would weaken the explicit role of `--schema` and `tool manifest` as the discovery contract |
| Treat `docs/*` link-check findings as real published-site defects | The maintainer confirmed the published site works, and the failures are consistent with raw filesystem validation on a symlinked mirror |
| Fix the remaining copied checklist link in `skills/cli-agent-evaluate/references/challenges/checklist.md` | The maintainer explicitly chose to leave it because the file is only a copy |
